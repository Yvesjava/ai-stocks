# AI股票分析系统 - 项目目标文档

## 一、项目概述

本项目旨在构建一套**多Agent协同工作的智能股票分析系统**，通过分工明确的多个AI Agent实现从新闻采集到股票推荐报告的全流程自动化分析。

## 二、系统架构

系统由5个核心Agent组成，形成**流水线式工作流程**：

### 2.1 Agent角色与职责

| Agent角色 | 主要职责 | 输出成果 |
|---------|--------|---------|
| **新闻采集员** | 定时从各渠道（财经网站、社交媒体、官方公告等）获取新闻和消息，进行要点整理和分类 | 结构化新闻摘要 |
| **股票收集员** | 分析新闻原料，识别提及或涉及的股票，初步筛选形成待分析股票列表 | 待分析股票清单 |
| **股票分析员** | 调用内网TradingAgents-CN、千问平台等深度分析工具，结合自身判断对股票进行二次筛选和分析 | 详尽分析报告 |
| **股票报告员** | 对分析后的股票进行第三次筛选，生成推荐股票的简报和摘要 | 最终推荐报告 |

### 2.2 工作流程

```
[新闻采集员] → [股票收集员] → [股票分析员] → [股票报告员]
    ↓              ↓             ↓             ↓
  新闻素材      待分析股票      分析报告      推荐简报
```

## 三、详细职责说明

### 3.1 新闻采集员（News Collector Agent）

**核心能力：**
- 定时任务触发（可配置时间间隔）
- 多渠道新闻采集：财经网站、新闻API、社交媒体、上市公司公告等
- 新闻清洗去重
- 关键信息提取：标题、来源、时间、摘要、涉及标的
- 新闻分类：宏观、行业、公司、事件等

**输出格式：**
```json
{
  "news_id": "xxx",
  "title": "新闻标题",
  "source": "新闻来源",
  "publish_time": "发布时间",
  "summary": "要点整理",
  "related_stocks": ["股票代码列表"],
  "category": "分类",
  "sentiment": "情绪判断"
}
```

### 3.2 股票收集员（Stock Collector Agent）

**核心能力：**
- 解析新闻采集员输出的结构化新闻
- 基于关键词、概念、事件等识别相关股票
- 初筛条件：相关性阈值、热度阈值等
- 维护待分析股票队列
- 股票基础信息补充（名称、板块、市值等）

**输出格式：**
```json
{
  "stock_code": "600519",
  "stock_name": "贵州茅台",
  "source_news": ["新闻ID列表"],
  "selection_reasons": ["入选原因"],
  "priority_score": 85,
  "timestamp": "收集时间"
}
```

### 3.3 股票分析员（Stock Analyst Agent）

**核心能力：**
- 调用内网部署的TradingAgents-CN进行深度技术面和基本面分析
- 调用千问平台（Qwen Finance API）进行财经领域深度研究
- 调用DeepSeek API进行综合评估分析
- 多平台分析结果汇总，计算共识推荐和置信度
- 技术面分析：K线、均线、成交量、技术指标等
- 基本面分析：财务报表、估值、盈利能力等
- 消息面分析：近期新闻、公告、研报等
- 综合研判，出具详细分析报告
- 二次筛选，形成重点推荐列表

### 3.4 股票报告员（Stock Reporter Agent）

**核心能力：**
- 解析股票分析员的分析报告
- 第三次筛选：复核分析质量，过滤逻辑不充分的股票
- 生成精炼的股票简报：核心逻辑、关键数据、风险提示
- 生成摘要：今日推荐、开仓逻辑、持有建议
- 格式化输出：支持Markdown、PDF、HTML等多种格式

**输出格式：**
```json
{
  "report_date": "报告日期",
  "recommended_stocks": [
    {
      "stock_code": "600519",
      "stock_name": "贵州茅台",

      "key_highlights": ["要点1", "要点2"],
      "risk_warning": "风险提示",
      "action_suggestion": "操作建议"
    }
  ],
  "summary": "今日整体市场简评和推荐总结"
}
```

## 四、技术要求

### 4.1 Agent框架要求
- 支持多Agent并行/串行执行
- 支持Agent间消息传递和状态共享
- 支持定时任务调度
- 支持日志记录和审计追踪

### 4.2 外部工具集成
- TradingAgents-CN（内网部署）：深度股票分析
- 千问平台（Qwen API）：财经领域深度研究
- 新闻数据源：可配置的多数据源接入

### 4.3 数据存储
- 新闻原始数据存储
- 中间处理结果存储
- 最终报告存储
- 支持历史查询和回溯

## 五、系统特点

1. **流水线作业**：各Agent职责单一，形成清晰的数据处理流水线
2. **三层筛选**：新闻→候选股→分析股→推荐股，逐层精选
3. **工具增强**：集成专业分析工具，提升分析深度
4. **可配置性**：各环节阈值、参数可调整
5. **可追溯**：完整记录每个环节的输入输出，便于审计
