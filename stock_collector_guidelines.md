# 股票收集员工作指南

## 1. 工作职责

### 1.1 核心职责
- **新闻数据分析**：分析新闻采集器提供的结构化新闻数据，提取相关股票信息
- **股票筛选**：根据预设规则对提取的股票进行筛选，识别具有投资价值的股票
- **股票评分**：对筛选出的股票进行多维度评分，计算优先级
- **数据存储**：将筛选和评分后的股票数据存储到指定文件中
- **数据更新**：定期更新股票数据，确保数据的及时性和准确性

### 1.2 具体任务
- **加载新闻数据**：从 `data/news/structured/news_database.json` 加载结构化新闻数据
- **提取股票信息**：从新闻中提取股票代码、名称、提及次数、情感倾向等信息
- **应用筛选规则**：
  - 最低提及次数：1次
  - 积极情感阈值：0.3
  - 关注的影响级别：high、medium
  - 最大时间间隔：7天
- **计算优先级分数**：
  - 基础分数：50分
  - 提及次数加分（最多30分）
  - 情感分数（最多15分）
  - 影响级别加分（high:10分, medium:5分）
  - 时间因素（最多10分）
- **去重处理**：基于股票代码去重，避免重复记录
- **数据排序**：按优先级分数降序排序
- **数据保存**：将处理后的数据保存到 `data/news/structured/stock_list.json`

## 2. 输出标准

### 2.1 数据格式
- **文件格式**：JSON
- **文件路径**：`data/news/structured/stock_list.json`
- **数据结构**：
  ```json
  [
    {
      "stock_code": "600519",
      "stock_name": "Guizhou Moutai",
      "source_news": ["NEWS_20260425120000_12345678"],
      "selection_reasons": ["公司公告：Guizhou Moutai released Q1 2026 report, revenue 100.5 billion yuan, YoY growth 15.2%"],
      "priority_score": 95.5,
      "mention_count": 3,
      "average_sentiment": 0.8,
      "impact_levels": ["high", "medium"],
      "latest_news_time": "2026-04-25T12:00:00",
      "timestamp": "2026-04-25T13:00:00"
    }
  ]
  ```

### 2.2 输出要求
- **完整性**：包含所有必要字段，无缺失信息
- **准确性**：股票代码、名称等信息准确无误
- **及时性**：最新新闻时间不超过7天
- **排序**：按优先级分数降序排列
- **去重**：无重复股票记录
- **数量**：每次运行至少输出5条有效股票记录

## 3. 验收标准

### 3.1 功能验收
- **数据加载**：成功加载新闻数据，无错误
- **股票提取**：正确提取新闻中的股票信息
- **筛选规则**：严格按照筛选规则过滤股票
- **评分计算**：优先级分数计算准确
- **数据存储**：数据成功保存到指定文件
- **去重处理**：无重复股票记录
- **排序正确**：股票按优先级分数降序排列

### 3.2 性能验收
- **运行时间**：单次运行时间不超过30秒
- **内存使用**：内存使用不超过500MB
- **稳定性**：连续运行10次无错误

### 3.3 质量验收
- **数据质量**：输出数据格式正确，字段完整
- **准确性**：股票信息和评分准确反映新闻内容
- **一致性**：相同输入产生相同输出
- **可追溯性**：每条股票记录可追溯到原始新闻

## 4. 工作流程

1. **启动**：由主系统调用 `StockCollector.collect()` 方法
2. **数据加载**：加载结构化新闻数据
3. **股票提取**：从新闻中提取股票信息
4. **筛选处理**：应用筛选规则，过滤不符合条件的股票
5. **评分计算**：计算股票优先级分数
6. **数据合并**：与现有数据合并，去重
7. **排序处理**：按优先级分数排序
8. **数据保存**：保存到指定文件
9. **完成**：返回处理结果

## 5. 异常处理

- **数据缺失**：当新闻数据文件不存在时，记录警告并返回空列表
- **数据格式错误**：当新闻数据格式错误时，记录错误并返回空列表
- **存储失败**：当保存数据失败时，记录错误并尝试重新保存

## 6. 维护与优化

- **规则调整**：根据市场变化和系统表现，定期调整筛选规则和评分算法
- **性能优化**：优化数据处理流程，提高运行效率
- **功能扩展**：根据业务需求，扩展股票分析维度和功能
