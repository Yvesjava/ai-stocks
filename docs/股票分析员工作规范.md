# 股票分析员（Stock Analyst Agent）工作规范

## 一、工作职责

### 1. 核心职责
- **数据获取**：从股票收集员生成的 `stock_list.json` 文件中读取待分析股票列表
- **多平台分析**：调用以下AI分析平台对每只股票进行深度分析：
  - TradingAgents-CN（内网部署）：技术面和基本面深度分析
  - 千问财经（Qwen Finance API）：财经领域专业研究
  - DeepSeek API：综合评估分析
- **报告汇总**：汇总三个平台的分析结果，计算共识推荐和置信度
- **数据存储**：将分析结果保存到指定文件路径，确保其他Agent可获取
- **异常处理**：处理API调用失败情况，使用模拟数据保证流程连续性

### 2. 工作流程
1. **加载股票列表**：读取 `data/news/structured/stock_list.json`
2. **加载新闻上下文**：读取 `data/news/structured/news_database.json` 获取相关新闻
3. **多平台分析**：对每只股票调用三个AI平台进行分析
4. **共识计算**：统计各平台推荐分布，计算加权共识结论
5. **结果保存**：
   - 保存最终分析结果到 `stock_final_analysis.json`
   - 生成兼容格式到 `stock_analysis.json`（供股票报告员使用）
   - 保存各平台原始报告到 `platform_reports/<stock_code>/` 目录

### 2. 存储路径
- **最终分析结果**：`data/news/structured/stock_final_analysis.json`
- **兼容格式**：`data/news/structured/stock_analysis.json`
- **各平台原始报告**：`data/news/structured/platform_reports/<stock_code>/`
  - 文件名格式：`{platform}_{datetime}.json`
  - 示例：`trading_agents_cn_20260425_103000.json`

### 3. 输出要求
- **完整性**：所有字段必须填写，无空值
- **一致性**：各平台报告格式统一，便于后续处理
- **时效性**：分析时间必须为当前时间
- **可读性**：JSON格式缩进2个空格，中文编码为UTF-8

## 三、验收标准

### 1. 功能验证
- **✓ 股票列表加载**：能正确读取 `stock_list.json` 文件
- **✓ 多平台分析**：能成功调用三个AI平台（或使用模拟数据）
- **✓ 共识计算**：能正确计算共识推荐和置信度
- **✓ 结果保存**：能生成所有指定的输出文件
- **✓ 异常处理**：API调用失败时能使用模拟数据继续流程

### 2. 数据完整性
- **✓ 字段完整性**：所有必要字段均有值
- **✓ 格式正确性**：JSON格式正确，编码无误
- **✓ 数据一致性**：不同格式文件中的数据一致
- **✓ 平台报告完整**：每个平台的原始报告都有保存

### 3. 兼容性验证
- **✓ 与股票收集员**：能正确读取 `stock_list.json`
- **✓ 与股票报告员**：生成的 `stock_analysis.json` 能被正确解析
- **✓ 与新闻采集员**：能正确加载 `news_database.json` 中的新闻上下文

### 4. 性能和可靠性
- **✓ 处理速度**：分析10只股票的时间不超过30秒
- **✓ 错误处理**：网络异常时能继续运行
- **✓ 日志记录**：详细记录分析过程和异常情况
- **✓ 去重处理**：避免重复分析同一股票

### 5. 质量指标
- **推荐一致性**：三个平台的推荐评级一致度应达到60%以上
- **置信度**：最终分析的置信度应达到0.7以上
- **数据准确性**：模拟数据应合理反映股票实际情况
- **文档完整性**：所有输出文件格式符合规范

## 四、API配置

### 环境变量配置

- 重试次数：1次
- 失败处理：使用模拟数据

## 六、故障排查

### 常见问题
1. **API调用失败**：检查网络连接和API密钥
2. **文件读取错误**：检查文件路径和权限
3. **JSON格式错误**：检查文件编码和格式
4. **性能问题**：减少同时分析的股票数量
