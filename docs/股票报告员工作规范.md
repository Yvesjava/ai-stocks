# 股票报告员工作规范

## 1. 角色定位

股票报告员是股票分析系统的**最终环节**，负责：
- 汇总各AI平台的分析结论
- 提炼核心观点，生成格式优美的推荐报告
- 通过多渠道（QQ、微信、企业微信等）通知用户
- 维护报告数据的关联性和可追溯性

---

## 2. 核心工作职责

### 2.1 职责清单

| 职责 | 描述 | 优先级 |
|------|------|--------|
| 报告生成 | 从分析结果中筛选优质股票，生成推荐报告 | P0 |
| 消息通知 | 通过多渠道发送报告通知 | P0 |
| 数据存储 | 按规范存储报告，确保数据可追溯 | P0 |
| 格式美化 | 生成格式规范、美观的报告文档 | P1 |
| 关联性分析 | 追踪数据从新闻到报告的演变 | P2 |

### 2.2 输入数据（读取文件）

股票报告员从以下文件读取数据：

| 文件路径 | 数据描述 | 格式 | 必填 |
|---------|---------|------|------|
| `data/news/structured/stock_analysis.json` | 股票分析结果 | JSON | 是 |
| `data/news/structured/stock_final_analysis.json` | 最终分析结果 | JSON | 否 |
| `data/news/structured/news_database.json` | 新闻数据库 | JSON | 否 |
| `data/news/structured/stock_list.json` | 股票列表 | JSON | 否 |

### 2.3 输出数据（写入文件）

| 文件路径 | 数据描述 | 格式 |
|---------|---------|------|
| `data/news/reports/{date}/stock_report_{timestamp}.json` | 报告数据 | JSON |
| `data/news/reports/{date}/stock_report.md` | Markdown报告 | Markdown |
| `data/news/structured/stock_report.json` | 最新报告（全局索引） | JSON |

### 2.4 通知发送

| 渠道 | 启用方式 | 消息格式 |
|------|---------|---------|
| 企业微信 | 环境变量配置 | Markdown |
| 微信 | 环境变量配置 | 文本 |
| QQ | 环境变量配置 | 文本 |

## 3. 数据结构定义

### 3.1 报告数据（JSON）

```json
{
  "report_date": "2026-04-25T10:30:00",
  "total_analyzed": 50,
  "total_stocks": 5,
  "recommended_stocks": [
    {
      "stock_code": "600519",
      "stock_name": "贵州茅台",
      "recommendation": "强烈推荐",
      "risk_level": "低",
      "target_price": 1800.0,
      "priority_score": 95,
      "brief": "核心逻辑简述...",
      "key_highlights": ["技术面金叉", "业绩增长"],
      "risk_warning": "市场波动风险...",
      "action_suggestion": "积极买入，逢低加仓"
    }
  ],
  "summary": "市场简评内容...",
  "generated_by": "Stock Reporter Agent"
}
```

### 3.2 股票简报结构

| 字段 | 类型 | 描述 | 必填 |
|------|------|------|------|
| stock_code | string | 股票代码，6位数字 | 是 |
| stock_name | string | 股票名称 | 是 |
| recommendation | string | 推荐评级 | 是 |
| risk_level | string | 风险等级 | 是 |
| target_price | number | 目标价格 | 是 |
| priority_score | number | 优先级分数，0-100 | 是 |
| brief | string | 核心逻辑 | 是 |
| key_highlights | array | 关键要点 | 否 |
| risk_warning | string | 风险提示 | 否 |
| action_suggestion | string | 操作建议 | 是 |

## 4. 工作流程

```
1. 读取分析结果
   └─> data/news/structured/stock_analysis.json

2. 执行最终筛选
   ├─> 推荐评级 >= Hold
   ├─> 风险等级 <= Medium
   ├─> 优先级分数 >= 50
   └─> 最多取10只股票

3. 生成股票简报
   ├─> 提取核心逻辑
   ├─> 识别关键要点
   ├─> 生成操作建议
   └─> 生成风险提示

4. 生成报告
   ├─> 构建报告数据结构
   ├─> 生成市场简评
   ├─> 生成Markdown报告
   └─> 保存到文件

5. 发送通知
   ├─> 格式化消息
   ├─> 发送到企业微信
   ├─> 发送到微信
   └─> 发送到QQ

6. 完成
```

## 7. 错误处理

### 7.1 错误类型

| 错误类型 | 处理方式 | 影响范围 |
|---------|---------|---------|
| 输入文件不存在 | 记录警告，返回空结果 | 终止执行 |
| JSON解析失败 | 记录错误，跳过该条数据 | 继续处理其他数据 |
| 筛选无结果 | 记录警告，生成空报告 | 继续执行 |
| 通知发送失败 | 记录错误，尝试下一渠道 | 记录失败渠道 |
| 文件写入失败 | 记录错误，抛出异常 | 终止执行 |

### 7.2 日志要求

- 日志文件：`data/news/logs/stock_reporter.log`
- 日志级别：INFO及以上
- 必记录项：
  - 执行开始和结束
  - 加载数据数量
  - 筛选结果数量
  - 通知发送结果
  - 错误详情

## 8. 与其他Agent的协作

### 8.1 数据依赖

| 上游Agent | 提供数据 | 文件路径 |
|----------|---------|---------|
| 新闻采集员 | 原始新闻数据 | data/news/raw/{date}/ |
| 股票分析员 | 分析结果 | data/news/structured/stock_analysis.json |

### 8.2 依赖条件

- 股票分析员必须先完成分析并输出结果
- stock_analysis.json必须存在且包含有效数据
- 数据文件必须使用UTF-8编码

### 8.3 数据共享方式

**方式1：文件系统共享**
- 报告生成后写入共享目录
- 其他Agent通过读取文件获取数据
- 使用文件锁机制避免并发冲突
- 配置示例：`reports_dir = "d:\ai-stocks\data\news\reports"`

**方式2：数据库直接访问**
- 报告数据可写入数据库供其他Agent访问
- 支持按时间、股票代码等条件查询

## 10. 监控与告警

### 10.1 监控指标

| 监控项 | 描述 | 告警阈值 |
|-------|------|---------|
| 报告生成成功率 | 成功生成报告的比率 | < 95% |
| 处理时间 | 单次执行耗时 | > 30秒 |
| 通知发送成功率 | 至少一个渠道成功发送 | < 100% |
| 数据完整性 | 必填字段缺失率 | > 1% |
| 文件存储延迟 | 从生成到存储的时间 | > 5秒 |

### 10.2 告警规则

| 告警级别 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 严重 | 分析结果文件不存在 | 立即通知，停止执行 |
| 警告 | 所有通知渠道发送失败 | 记录日志，发送告警 |
| 提示 | 筛选结果为空 | 记录警告，生成空报告 |

## 11. 验收标准

### 11.1 功能验收

| 验收项 | 验收条件 | 验证方法 |
|-------|---------|---------|
| 报告生成 | 成功生成JSON和Markdown格式报告 | 检查文件存在 |
| 数据完整性 | 报告包含所有必填字段 | JSON Schema验证 |
| 筛选正确 | 筛选结果符合筛选规则 | 逻辑验证 |
| 通知发送 | 至少一个渠道发送成功 | 日志检查 |
| 文件存储 | 报告正确保存到指定目录 | 文件检查 |

### 11.2 数据验收

| 验收项 | 验收条件 | 检查方法 |
|-------|---------|---------|
| 股票代码 | 非空，符合6位数字格式 | 正则验证 |
| 推荐评级 | 为预定义评级之一 | 枚举验证 |
| 风险等级 | 为预定义等级之一 | 枚举验证 |
| 目标价格 | 大于0的数值 | 数值验证 |
| 优先级分数 | 0-100之间的数值 | 范围验证 |

### 11.3 质量验收

| 验收项 | 验收条件 | 检查方法 |
|-------|---------|---------|
| 报告可读性 | Markdown格式正确，无乱码 | 人工检查 |
| 消息简洁性 | 通知消息不超过500字 | 长度检查 |
| 处理时间 | 单次执行不超过30秒 | 时间记录 |
| 错误处理 | 异常情况有明确提示 | 日志检查 |
